페이지 구성

학과 규정에 맞추어 연구 동기, 연구 내용, 결과 및 성능, 참고문헌, 관련 사이트, 추가 자료를 각각 별도 페이지로 구성하였다. 전체 레이아웃은 복잡한 효과 없이 간결한 문서형 형식을 유지하도록 설계하였다.

01. 연구 동기 02. 연구 내용 03. 결과 및 성능 04. 참고문헌 및 관련 사이트 05. 논문 및 부가 자료

작성 메모

  • 메인 페이지에는 제목, 저자, 소속, 지도교수, 초록을 배치하였다.
  • 하위 페이지는 각각 독립적인 문서처럼 읽히도록 구성하였다.
  • 대표 이미지는 아직 넣지 않았으며, 필요 시 연구 내용 또는 결과 페이지에 추가할 수 있다.

현재 GitHub Pages 배포용 파일은 모두 docs/ 폴더 기준으로 관리되고 있다.
